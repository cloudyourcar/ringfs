1 # RingFS, a small Flash-based ring buffer.
2 
3 RingFS is a persistent, Flash-based ring buffer designed for embedded software.
4 It's aimed at storing non-critical data that can be expunged on the FIFO basis
5 as needed. Typical uses include:
6 
7 * telemetry data,
8 * debug logs,
9 * stack traces.
10 
11 RingFS has been designed to run on NOR Flash memory, which exhibits the following
12 semantics:
13 
14 1. Bits are programmed by flipping them from 1 to 0 with byte granularity.
15 2. Bits are erased by flipping them from 0 to 1 with sector granularity.
16 
17 ## Features
18 
19 * Designed and optimized for NOR Flash memory.
20 * Stores fixed-size objects in a FIFO buffer.
21 * Written in ISO C99.
22 * No dynamic memory allocation.
23 * Basic robustness features for error recovery.
24 
25 ## Usage
26 
27 1. Add ``ringfs.[ch]`` to your project.
28 2. Implement the required Flash ops (``sector_erase``, ``program``, ``read``).
29 3. Glue your Flash ops with ringfs using ``struct ringfs_flash_partition``.
30  See ``tests.c`` for an example.

36 ## Non-Features
37 
38 The ring buffer has been designed to be as simple as possible. Therefore, the
39 following are non-features that will *not* be implemented:
40 
41 * Variable object sizes (makes things much more complicated).
42 * Complicated error recovery (we can lose data in edge cases).
43 * Upgrades (complex, also unnecessary in our use cases).
44 
45 Actually, on the second thought, I may consider adding support for variable
46 object sizes some day.
